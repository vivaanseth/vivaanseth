## About Me

I am **Vivaan**, a student developer and product builder interested in creating software that feels useful, thoughtful, and genuinely original. I focus on the intersection of **artificial intelligence, desktop applications, product engineering, UI/UX, and robotics**.

My flagship project is **Luminal**, a local-first desktop work-memory and momentum tool. I am also preparing **Jarvis**, a free and open-source desktop assistant, for public release. It is designed to remember the context behind unfinished computer-based work and help people return without reconstructing everything they were doing. I am especially interested in using AI as a supporting ingredient inside strong products—not as a replacement for the product itself.

Beyond software, I have worked across multiple seasons of **VEX IQ Robotics**, contributing through engineering documentation, organization, design thinking, and competition preparation. Building products and robots has taught me to break large problems into systems, test ideas quickly, and improve them through evidence.

### Open To

- Student hackathons and build challenges
- Open-source collaboration
- Product feedback and technical mentorship
- Building with other motivated student developers
- Interesting software, AI, design, and robotics projects

## Featured Projects

<details open>
<summary><strong>Luminal — Local-First Work Memory</strong></summary>

<br />

> A desktop work-memory and momentum tool that remembers the context behind unfinished work and helps people return without reconstructing what they were doing.

| Category | Details |
|---|---|
| **Stack** | Desktop application development, AI integrations, local context systems, product design |
| **Status** | Active development |
| **Product Focus** | Restoring context and continuity across unfinished computer-based work |
| **AI Role** | Helps organize and surface useful context rather than acting as a generic chatbot |
| **Privacy** | Local-first direction with privacy-conscious product decisions |
| **Key Learning** | Desktop architecture, context design, product positioning, UI systems, and responsible AI integration |
| **Repository** | Private during active development |

Luminal explores a problem that many productivity tools ignore: returning to a project often requires rebuilding the mental thread behind it. Instead of scoring users, monitoring employees, or turning work into a game, Luminal focuses on quietly preserving useful context.

**Core directions**

- Remember the thread behind unfinished work
- Restore relevant application and project context
- Reduce the friction of switching back into deep work
- Keep the experience calm, private, and unobtrusive
- Use AI selectively where it improves understanding or retrieval

</details>

<details>
<summary><strong>Jarvis — Open-Source Desktop Assistant</strong></summary>

<br />

> A free and open-source personal desktop assistant designed to make everyday computer tasks easier through one unified interface.

| Category | Details |
|---|---|
| **Stack** | Desktop application development, automation, AI integrations, interface design |
| **Status** | Preparing for public open-source release |
| **Product Focus** | Helping users interact with their computer and complete common tasks more naturally |
| **AI Role** | Interprets requests, supports task execution, and makes the assistant more flexible |
| **Open Source** | Free to use, inspect, modify, and build upon |
| **Key Learning** | Desktop automation, application architecture, tool integration, safety boundaries, and designing a useful assistant |
| **Repository** | Public repository coming soon on [github.com/vivaanseth](https://github.com/vivaanseth?tab=repositories) |

Jarvis is a personal-assistant project built around a simple idea: computer assistance should feel like a real application, not just a chat window. The project brings commands, tools, and useful information into a single desktop experience.

**Project goals**

- Launch and behave like a normal desktop application
- Support natural text-based interaction
- Connect useful computer actions through a unified assistant
- Keep the project free and accessible
- Make the code open source so others can learn from it and contribute
- Add capabilities carefully rather than promising unrestricted control

Once the repository is published, this section can link directly to it and include installation instructions, supported features, and contribution guidelines.

</details>
